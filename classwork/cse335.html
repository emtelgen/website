Sorting Algorithms

Selection Sort

Selection sort iterates through the array, and fills the smallest unsorted item in the largest sorted spot. Time Complexity is O(n²) and the space is O(1).

Insertion Sort

This sorting algorithm iterates through the array, and fills the next unsorted item into it's correct position within the sorted items of the array. Time Complexity is at worst O(n²) and at best O(n).

Hybrid Merge Sort

This is a hybrid of the normal merge sort, which utilizes merge sort up until a certain threshold, at which it will begin to use insertion sort. This aids the runtime because insertion sort tends to run at faster complexities than merge sort when the array length is small, giving this hybrid sort an advantage. The time complexity of this sort is O(nlogn).

Coding Portion

Unit Coding Challenge

This coding challenge worked through four different ways to find the indices of the first two numbers that sum up to the target number. We utilized a sorted array with two pointers to the current right and left indices, and incremented them based on the current value of their sum. This was the fastest way to utilize two-sum with a sorted array, with a time complexity of O(n). Other ways involve brute-force adding every pair of numbers possible with a time complexity of O(n²), as well as utilizing a dictionary to store numbers needed to add up to the sum, which has a time complexity of O(n), but also uses O(n) space for the dictionary.

Unit Project

This project employed the previously discussed sorting algorithms, to take individual SAT scores and return information on which parts of the score were above the median parameter. Utilizing the hybrid merge sort that we implemented, the time complexity of this function is O(nlogn).

Tree Traversals

Inorder Traversal

Follows the search pattern of: left child, then root, then right child. This will start at the very left-most element and traverse the roots and right children, eventually hitting the root node, and then traversing to the root node's right children.

Preorder Traversal

Follows the search pattern of: root, left child, right child. This will start at the root and go down all the way until the left-most node is hit, and then traverse backwards through each root node's right child, eventually getting back to the root node and traversing through the right children.

Postorder Traversal

Follows the search pattern of: left child, right child, root. This will start at the very left-most node and traverse back up until hitting the root, then traversing through all the right children before the root node is added.

Levelorder Traversal

This traverses through each level of the tree, yielding each node based on it's height in the tree, smaller heights being added first. It utilizes a queue to keep track of the current level nodes children.
