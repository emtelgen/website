Sorting and Divide-and-Conquer

Unit Notes

Sorting Algorithms

Selection Sort

Selection sort iterates through the array, and fills the smallest unsorted item in the largest sorted spot. Time Complexity is O(n²) and the space is O(1).

Insertion Sort

This sorting algorithm iterates through the array, and fills the next unsorted item into it's correct position within the sorted items of the array. Time Complexity is at worst O(n²) and at best O(n).

Hybrid Merge Sort

This is a hybrid of the normal merge sort, which utilizes merge sort up until a certain threshold, at which it will begin to use insertion sort. This aids the runtime because insertion sort tends to run at faster complexities than merge sort when the array length is small, giving this hybrid sort an advantage. The time complexity of this sort is O(nlogn).

Unit Coding Challenge

This coding challenge worked through four different ways to find the indices of the first two numbers that sum up to the target number. We utilized a sorted array with two pointers to the current right and left indices, and incremented them based on the current value of their sum. This was the fastest way to utilize two-sum with a sorted array, with a time complexity of O(n). Other ways involve brute-force adding every pair of numbers possible with a time complexity of O(n²), as well as utilizing a dictionary to store numbers needed to add up to the sum, which has a time complexity of O(n), but also uses O(n) space for the dictionary.

Tree Traversals

Inorder Traversal

Follows the search pattern of: left child, then root, then right child. This will start at the very left-most element and traverse the roots and right children, eventually hitting the root node, and then traversing to the root node's right children.

Preorder Traversal

Follows the search pattern of: root, left child, right child. This will start at the root and go down all the way until the left-most node is hit, and then traverse backwards through each root node's right child, eventually getting back to the root node and traversing through the right children.

Postorder Traversal

Follows the search pattern of: left child, right child, root. This will start at the very left-most node and traverse back up until hitting the root, then traversing through all the right children before the root node is added.

Levelorder Traversal

This traverses through each level of the tree, yielding each node based on it's height in the tree, smaller heights being added first. It utilizes a queue to keep track of the current level nodes children.

Graphs and Graph Algorithms

Unit Notes

Algorithms

Depth-First Search

Implementation of a recursive Depth First Search traversal through a graph, that starts at the beginning node and traverses one path until it ends, recursively entering previous nodes in the path to find other paths. The run time of this is O(N+M) where N is the number of edges and M is the number of vertices.

Breadth-First Search

This traversal matches Depth-First Search in runtime, but instead of finding the deepest paths first, it will visit all neighbors of the current node and traverse to neighbors of each neighbor before moving to the next neighbor.

Floyd-Warshall Transitive Closure

This algorithm steps through each node in the group, and adds an edge between two vertices if any of the neighbors of the current vertex contain that edge. Essentially, if A goes to B and B goes to C, then A goes to C. The runtime of this function is O(n3)

Greedy Algorithms and Dynamic Programming

Unit Notes

Greedy Algorithms

Djikstra's Algorithm

Djikstra's Algorithm finds the shortest path between two vertices by keeping a table of the weights between each vertex, and at each step choosing the next minimum available step. This is a greedy algorithm that chooses the next best option based on the previous options and runs in O((N*M)logN).

A-Star Search Algorithm

A-Star search is similar to Djikstra's in that it finds the shortest path between two vertices, however it adds a heuristic calculation to increase decision accuracy and help the algorithm make the choice quicker. It also runs in O(N*M(logN))

Prim's Minimum Spanning Tree

This algorithm finds a minimum spanning tree for a weighted and and undirected graph by starting at one vertex and continuously adding choosable edges for each minimum edge chosen. This runs in O(N+M(logN)).

Kruskal's Minimum Spanning Tree

This algorithm finds a minimum spanning tree for a weighted and and undirected graph by continuously choosing the minimum edge currently existing in the graph, ensuring that no cycle is created if an edge is chosen. This runs in O(N+M(logN)).

Dynamic Programming

House Robber

The house robber problem is a dynamic programming problem that involves determining which houses a robber should raid in order to maximize profits. Iterating through each house robbery amount, the robber chooses whether to rob the house or not based on the current maximum profit. This runs in O(N) time.

Maximum Sum

This dynamic programming function chooses the maximum sub of non-adjacent values in an array by taking the max of the previous element, and the current element added with the element preceding the previous element. This runs in O(N) time.
