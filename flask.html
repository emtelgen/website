Data Analysis with Web Servers and Database Storage

The project I worked with a team on while employed by Michigan State University St. Andrews. We were tasked to create a website that took data from temperature and humidity sensors that were located across campus and exported that data through a website that we created.

Creation Procedure

Step One

Using Bootstrap functionality, we created the front-end of the website for users. This included a navigation bar with login and user details as well as access to separate areas of the university. One could click on individual rooms with buttons that were located on a map, which led to a new tab that displayed graphs for the data in that room.

Step Two

Displaying graphs using the data from the temperature and humidity sensors. These graphs were created using JavaScript and continuously updated based on the data from the sensors that constantly updated.

Step Three

Creating a database using MySQL that served to store all the information for the users, rooms, and sensors. The user database distinguished admins from normal users, who had greater access to the data in the website.

Step Four

Taking input from the user with a login form that could be accessed. This would connect to the database, determine the login validity, and allow access to information on the website based on the outcome.

Additional Modifications

Additional modifications that could be used to make this website better would include more additions to the HTML used to make the website more visually appealing. Other aspects that could be added would be increasing the functionality of the graphs with more user interaction.
