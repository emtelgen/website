Web Scraping and Interaction

Using the Python selenium library, determined validity of user reports based on a created algorithm for each type of report. Returned the report with the reviewed verdict with 95% accuracy. Click the button below for the link to the full code on GitHub.

Creation Procedure

Step One

Accessing the web server in the main script. This was done using a driver to access the Website and continuously allowing access after reloading.

Step Two

Extracting the information from each report and formatting it (function shown was used to extract the username within the report). This was executed in the main script using the functions in the supplementary file "report_info.py".

Step Three

Running an algorithm based on the report type. The main script would access supplementary scripts in sibling failures based on the information extracted in the previous step. The picture shows an example of accessing these functions.

Step Four

Processing judgement on the report based on the returned information. The sleep function is added to not overload the website with constant judgement votes.
